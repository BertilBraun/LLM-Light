# Running LLM-Light

This page collects the commands needed to install, test, run, evaluate, generate
from, and export LLM-Light experiments.

## Environment

LLM-Light targets Python 3.10+ and PyTorch 2.x.

Editable install with development tools:

```bash
python -m pip install -e ".[dev]"
```

Equivalent `uv` workflow:

```bash
uv sync --extra dev
```

Optional synthetic-data generation dependencies:

```bash
python -m pip install -e ".[generation]"
```

With development tools and generation dependencies under `uv`:

```bash
uv sync --extra dev --extra generation
```

GPU training uses your installed PyTorch build. For multi-GPU runs, install a
CUDA-enabled PyTorch build compatible with the host before launching `torchrun`.

## TinyPython Corpus Generation

The public `BertilBraun/TinyPython` Hugging Face dataset was generated by this
project with:

```text
llm_lite/scripts/generate_tinypython.py
```

The generator uses local vLLM teacher models, a controlled semantic seed space,
stochastic samples per seed, Python parsing/validation, resumable JSONL output,
and separate invalid-output records with rejection reasons.

Small pilot:

```bash
uv run python -m llm_lite.scripts.generate_tinypython \
  --model Qwen/Qwen2.5-Coder-7B-Instruct \
  --num-seeds 500 \
  --samples-per-seed 2 \
  --batch-size 512 \
  --output data/tinypython/qwen25_pilot.jsonl
```

Larger teacher run shape:

```bash
uv run python -m llm_lite.scripts.generate_tinypython \
  --model Qwen/Qwen2.5-Coder-7B-Instruct \
  --num-seeds 500000 \
  --samples-per-seed 2 \
  --batch-size 512 \
  --output data/tinypython/qwen25_coder_7b.jsonl
```

The script writes invalid generations to
`data/tinypython/qwen25_coder_7b.invalid.jsonl` by default.

Fresh-instance helper:

```bash
bash scripts/generate_tinypython_dataset.sh
```

Full public-corpus-scale helper run:

```bash
RUN_FULL=1 bash scripts/generate_tinypython_dataset.sh
```

The helper defaults to two teacher models, two samples per seed, 500 pilot
seeds, and 500,000 full-run seeds. With two teachers and two samples per seed,
full mode requests up to 2,000,000 raw generation attempts.

## Tests and Static Checks

Run all tests:

```bash
python -m pytest
```

Run a faster smoke subset when iterating on pipeline behavior:

```bash
python -m pytest tests/test_pipeline_stages.py tests/test_training_checkpoint.py
```

Run formatting and lint checks:

```bash
python -m ruff format --check .
python -m ruff check .
```

## Smoke Pipeline

The smallest complete run trains a character-level dense GPT to reproduce
`hello world\n`:

```bash
python -m llm_lite.scripts.run_pipeline \
  --config configs/verify_one_sentence.yaml
```

Review the planned stages without execution:

```bash
python -m llm_lite.scripts.run_pipeline \
  --config configs/verify_one_sentence.yaml \
  --dry-run
```

Force recomputation from a stage and all downstream stages:

```bash
python -m llm_lite.scripts.run_pipeline \
  --config configs/verify_one_sentence.yaml \
  --force pretraining
```

Run only part of the pipeline:

```bash
python -m llm_lite.scripts.run_pipeline \
  --config configs/verify_one_sentence.yaml \
  --from tokenizer \
  --to pretraining
```

Useful integration smoke configs:

```bash
python -m llm_lite.scripts.run_pipeline \
  --config tests/configs/verify_local_text.yaml

python -m llm_lite.scripts.run_pipeline \
  --config tests/configs/verify_byte_bpe.yaml
```

Distributed CPU smoke, using two local `gloo` processes:

```bash
torchrun --standalone --nproc_per_node=2 \
  -m llm_lite.scripts.run_pipeline \
  --config tests/configs/distributed_data_parallel_smoke.yaml \
  --force
```

Some Windows CPU-only PyTorch builds can fail in the launcher before project
code starts because of `TCPStore` or `use_libuv` support. Linux GPU/NCCL
training should use the same `torchrun` shape.

## TinyPython MoE Pipeline

The main Python MoE config is:

```text
configs/python_moe_full.yaml
```

It streams `BertilBraun/TinyPython` from Hugging Face, formats each record as:

```text
{task_description}

{code}
```

and trains a top-1 MoE GPT with a 6000-token Rust byte-level BPE tokenizer.

Prepare all data artifacts without starting pretraining:

```bash
python -m llm_lite.scripts.run_pipeline \
  --config configs/python_moe_full.yaml \
  --to packed_dataset
```

Launch the distributed pretraining stage on two GPUs:

```bash
CUDA_VISIBLE_DEVICES=0,1 torchrun --standalone --nproc_per_node=2 \
  -m llm_lite.scripts.run_pipeline \
  --config configs/python_moe_full.yaml \
  --from pretraining \
  --to pretraining
```

With `uv`:

```bash
CUDA_VISIBLE_DEVICES=0,1 uv run torchrun --standalone --nproc_per_node=2 \
  -m llm_lite.scripts.run_pipeline \
  --config configs/python_moe_full.yaml \
  --from pretraining \
  --to pretraining
```

The committed config represents the first training phase:

```text
maximum_steps: 3750
learning_rate: 0.001
```

The recorded TinyPython MoE result used staged training phases:

```text
3,750 steps at LR 0.001
7,500 steps at LR 0.0005
10,000 steps at LR 0.00025
```

Changing `training.maximum_steps`, optimizer settings, batch size, precision,
gradient clipping, model configuration, seed, or distributed configuration now
creates a new pretraining artifact. Interrupted runs resume only when the
recomputed pretraining fingerprint is unchanged.

The completed TinyPython MoE artifact bundle is published as the
[Python-Run-V1 GitHub release](https://github.com/BertilBraun/LLM-Light/releases/tag/Python-Run-V1).
Use that release bundle when inspecting the recorded run on a checkout that does
not have the full `runs/python_moe_full` directory locally.

For fresh GPU instances, `scripts/train.sh` wraps the same stage sequence for
any config. `CONFIG_PATH` is required:

```bash
CONFIG_PATH=configs/python_moe_full.yaml \
NPROC_PER_NODE=2 \
GPU_IDS=0,1 \
bash scripts/train.sh
```

Useful overrides:

```bash
PREPARE_DATA=0 RUN_EVALUATION=0 CONFIG_PATH=configs/python_moe_full.yaml bash scripts/train.sh
```

## Evaluation

Run the configured evaluation stage for an existing run:

```bash
python -m llm_lite.scripts.run_pipeline \
  --config configs/python_moe_full.yaml \
  --from evaluation \
  --to evaluation \
  --force evaluation
```

The Python MoE config evaluates:

- Validation perplexity on the validation split.
- Python completion tasks from
  `tests/fixtures/tinypython_completion/tasks.jsonl`.

Python completion evaluation trims trailing newlines from signature prompts
before generation. It counts parse success, subprocess execution success, check
passes, total checks, and pass rate.

Training-time evaluation is scheduled from checkpoint events when
`training.evaluation` is configured and `run_plan --max-parallel-jobs` leaves a
free job slot. Checkpoint evaluation artifacts are written under:

```text
artifact_store/evaluation/<checkpoint-evaluation-fingerprint>/
```

and mirrored into the run TensorBoard view.

## Generation

Generate from the latest checkpoint of a completed run:

```bash
python -m llm_lite.scripts.generate \
  --config configs/python_moe_full.yaml \
  --prompt "Given a list of integers, write a function that returns the sum of all even numbers in the list." \
  --maximum-new-tokens 160
```

Include the prompt in the printed output:

```bash
python -m llm_lite.scripts.generate \
  --config configs/python_moe_full.yaml \
  --prompt "Given a string, write a function that returns the string in reverse order." \
  --maximum-new-tokens 160 \
  --include-prompt
```

Generation uses the `inference` block in the config unless overridden by CLI
arguments. Current configs use KV-cache inference for normal generation.

## TensorBoard

Run-level pipeline summaries are written to:

```text
runs/<experiment>/tensorboard
```

These summaries use one text table per completed pipeline stage. The raw,
processed, tokenizer, and packed-dataset tables include dataset document and
byte counts, tokenizer vocabulary/training counts, packed sequence and token
counts, pad fraction, average sequence fill, worker counts, and stage duration.

Inspect pretraining curves:

```bash
python -m tensorboard.main \
  --logdir runs/python_moe_full/tensorboard/pretraining
```

Pretraining TensorBoard includes loss, learning rate, gradient norm, local and
distributed throughput, distributed world size, checkpoint timing, and
training-time evaluation scalars when configured.

MoE runs also include router summaries under `moe/`. Each router layer writes
selected-expert histograms, compact layer summaries for expert usage mean,
standard deviation, minimum, maximum, normalized entropy, imbalance, and
dominance, plus `moe/summary` worst-layer imbalance, dominance, and entropy
scalars. These summaries are intended to make expert collapse visible without
adding many raw plots.

Final evaluation TensorBoard is written to:

```text
runs/<experiment>/tensorboard/evaluation
```

Evaluation scalars include perplexity loss, perplexity, document and sequence
counts, exact-reproduction status, fixed-prompt sample count, and Python
completion task count, parsed count, executed count, passed checks, total
checks, pass rate, and per-task-family summaries when the task file provides
`task_family`.

For the one-sentence smoke run:

```bash
python -m tensorboard.main \
  --logdir runs/verify_one_sentence/tensorboard/pretraining
```

## Exporting Run Bundles

Do not commit full run directories, checkpoints, packed shards, or TensorBoard
event logs to source control. Export a compact bundle:

```bash
python -m llm_lite.scripts.export_run_bundle \
  --run-dir runs/python_moe_full \
  --output python_moe_full_bundle.zip
```

The default bundle includes:

- `resolved_config.json`
- `pipeline.jsonl`
- `performance.jsonl`, when present
- tokenizer artifact files
- pretraining manifest and metrics
- training-evaluation logs
- final evaluation manifest and report, when present
- latest checkpoint only
- TensorBoard event files
- `bundle_manifest.json`

The published TinyPython MoE result is available as the `Python-Run-V1` GitHub
release bundle. Full run directories are intentionally kept out of source
control.

Exclude TensorBoard event files:

```bash
python -m llm_lite.scripts.export_run_bundle \
  --run-dir runs/python_moe_full \
  --output python_moe_full_bundle.zip \
  --no-include-tensorboard
```

Include every checkpoint instead of only the latest checkpoint:

```bash
python -m llm_lite.scripts.export_run_bundle \
  --run-dir runs/python_moe_full \
  --output python_moe_full_all_checkpoints.zip \
  --include-all-checkpoints
```

## Packed Dataset Benchmark

Compare packed dataset access modes:

```bash
python -m llm_lite.scripts.benchmark_packed_datasets
```

The script is a local performance probe, not a correctness requirement for
normal training.
